Hexo的安装以及Yilia主题配置

前言

最近研究天池算法看到了Wepon的博客非常好看。于是看到底部的Hexo Theme Yilia by Litten，想到了这是一个基于github.io的博客，域名用的是个性域名，框架使用的但是Hexo，主题用的是Litten的Yilia。早就想有一个自己的博客，于是。。。

1.安装noed.js

执行nodejs -v出现版本号则安装成功

2.安装hexo-1

用npm安装并执行初始化

这时出现/usr/bin/env: node: 没有那个文件或目录
于是百度了一下是nodejs和node名字问题，重新连接一下
sudo ln -s /usr/bin/nodejs /usr/bin/node
这次再执行hexo init blog初始化成功。

3.安装Yilia主题

在博客的根目录下进行主题克隆

下面修改配置文件_config.yml最下面几行
特别注意：后面有空格

同时在前面还可修改博客名称等信息

4.安装hexo-2

这时出现ERROR Plugin load failed: hexo-renderer-stylus
于是

再次hexo server打开http://localhost:4000,即可看到自己的博客。

5.配置github

  1. 申请账号
  2. 建立仓库name.github.io
  3. 配置
    • 首先在本地创建 ssh key：
      ssh-keygen -t rsa -C "your_email@youremail.com"
      后面的 your_email@youremail.com 改为自己的邮箱，之后会要求确认路径和输入密码，这里使用默认的一路回车就行。
    • 成功的话会在 ~/ 下生成 .ssh 文件夹，打开 id_rsa.pub，复制里面的 key，回到 github，进入 settings，左边选择 SSH keys，Add SSH Key，title 随便填，粘贴 key。
      为了验证是否成功，在 git bash 下输入：
      ssh -T git@github.com
      如果是第一次的会提示是否 continue，输入 yes 就会看到：You’ve successfully authenticated, but GitHub does not provide shell access，这就表示已成功连上 github。
    • 接下来我们要做的就是把 github 上面建立的仓库克隆到本地，在此之前还需要设置 username 和 email，因为 github 每次 commit 都会记录他们。

6.生成静态页面和发布

每次文章修改后都需要重新生成静态文件并发布

此时提示ERROR Deployer not found: github
下面执行npm install hexo-deployer-git --save
将deploy 的 type由github改为git即可
